# Codex 七天额度

本上下文描述 Codex 七天额度伴随悬浮条所使用的统一业务语言，重点区分额度周期、数据新鲜度与演示数据。

## Language

**七天额度**：
严格持续 10080 分钟的 Codex 额度限制，不等同于按自然周划分的额度。
_Avoid_: 周额度、自然周额度

**当前额度周期**：
当前生效的七天额度时间区间；开始时刻属于该周期，重置时刻不属于该周期。
_Avoid_: 本周、当前自然周

**当前使用量**：
当前额度周期内已经消耗的额度百分比。
_Avoid_: 剩余额度、可用百分比

**新鲜额度数据**：
最近一次读取成功、尚未到达重置时刻且包含有效使用量与周期时间的额度数据。

**部分额度数据**：
包含有效当前使用量，但缺少周期时间等非核心信息的额度数据。

**陈旧额度数据**：
当前额度周期尚未结束，但最近一次刷新失败，因此只能沿用上一次成功读取结果的数据。

**上一周期数据**：
额度周期已经结束、不能再代表当前额度状态，但仍可作为参考保留的数据。

**可重置次数**：
Codex 当前报告的可用额度重置次数；它不是已经完成的重置历史。

**演示数据**：
仅用于浏览器界面预览、不得被解释为用户真实账号额度的数据。
_Avoid_: 模拟真实额度、当前额度
