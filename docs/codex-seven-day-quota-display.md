# Codex 七天额度窗口展示逻辑

本文记录 2026-07-24 达成的展示逻辑共同理解。它是后续实现与验收依据，本次仅记录设计，不修改应用行为。

## 范围

本逻辑覆盖额度窗口选择、周期推导、使用量、日期格式、悬停详情、数据状态和刷新行为。

设置页、自动更新、窗口拖动和吸附行为保持现状。胶囊扩宽时，只同步调整依赖悬浮条宽度的配置与吸附计算。

## 数据选择

通过 Codex App Server 的 `account/rateLimits/read` 读取额度：

1. 顶层 `rateLimits` 存在且不为 `null` 时，只检查该对象。
2. 顶层 `rateLimits` 缺失或为 `null` 时，回退到 `rateLimitsByLimitId["codex"]`。
3. 顶层对象存在但没有七天窗口时，不使用明细回退。
4. 只接受 `windowDurationMins === 10080` 的窗口。
5. 选定对象中必须恰好有一个匹配窗口；零个表示“未找到七天额度”，多个表示“额度数据异常”。
6. 其他 `limitId` 暂不展示。

原生应用读取失败时不得回退到 Mock 数据。

## 数据校验与周期推导

- `usedPercent` 必须是 `0–100` 范围内的有效数字。缺失、非数字或超出范围均判定为数据不完整或异常，不做截断。
- `resetsAt` 是当前额度周期的右侧开放边界。
- 有效开始时刻按 `resetsAt - 10080 × 60 秒` 推导。
- 当前额度周期采用 `[开始时间, resetsAt)`。
- 先按绝对秒数推导，再将两端转换为操作系统本地时区。跨越夏令时时不按本地日历补偿。
- 缺少有效 `resetsAt` 时，仍可展示有效使用量，但周期时间标记为未知。
- 首次读取到已经过期的 `resetsAt` 时，不得将该数据冒充为当前周期。

## 正常展示

胶囊保持单行，目标窗口宽度约为 360px、内容宽度约为 346px，最终尺寸以 macOS 和 Windows 的实际字体渲染为准。

主内容固定包含：

- 表示已用比例的进度条；
- 明确文案 `已用 N%`；
- 紧凑本地日期范围，如 `7月18日–7月25日`。

跨年时仍使用 `12月29日–1月5日` 这类紧凑格式，不在胶囊内加入年份。

中间的“进度条 + 已用比例 + 日期范围”是悬停详情触发区，并且支持键盘聚焦。刷新和设置按钮保留各自提示。

悬停详情固定包含：

- 完整本地周期，如 `2026年7月18日 15:28 → 2026年7月25日 15:28`；
- `已使用：N%`；
- `可重置：N 次`，仅在服务端提供该字段时显示；明确返回 `0` 时必须显示；
- 最后一次成功同步的完整本地时间。

普通用户界面不显示 `limitId`、数据源等技术字段。

## 展示状态

| 状态 | 胶囊主内容 | 状态点 | 悬停详情 |
| --- | --- | --- | --- |
| 首次加载 | `正在读取七天额度…`，轻量加载动画 | 不显示 | 无额度值 |
| 新鲜数据 | `已用 N% · 开始日期–重置日期` | 绿色 | 完整周期、使用量、可重置次数、最后同步时间 |
| 刷新中且已有数据 | 保留原数据，刷新按钮显示忙碌状态 | 黄色 | 标明正在刷新 |
| 周期内刷新失败 | 保留上一次成功数据 | 黄色 | 标明更新失败、最后成功同步时间及归类后的原因 |
| 缺少重置时间 | `已用 N% · 周期时间未知` | 黄色 | 说明 Codex 未返回重置时间 |
| 周期到点刷新中 | `已用 N% · 周期已结束，正在更新` | 黄色 | 保留旧周期完整边界 |
| 到点刷新仍失败 | `已用 N% · 上一周期 开始日期–结束日期` | 黄色 | 明确这是上一周期数据并给出失败原因 |
| 首次读取失败 | `额度读取失败` | 红色 | 用户可理解的归类原因 |
| 未找到匹配窗口 | `未找到七天额度` | 红色 | 说明 Codex 未返回七天窗口 |
| 数据无效或歧义 | `额度数据异常` | 红色 | 说明返回数据不完整或存在歧义 |
| 浏览器预览 | 明确显示 `演示数据` | 灰色 | 说明此数据仅用于界面预览 |

错误文案使用“未登录 Codex”“找不到 Codex CLI”“读取超时”“未返回七天额度”等用户可理解的分类。原始 JSON、内部路径和进程错误只进入本地诊断日志。

所有加载与状态变化应向辅助技术提供适当语义；首次加载使用 `aria-live="polite"`，无有效百分比时不得暴露伪造的 `0%` 进度值。

## 颜色语义

状态点只表达数据可信度，不表达消耗程度。

进度条只表达额度消耗风险：

- `0%–79%`：蓝绿色；
- `80%–99%`：橙色；
- `100%`：红色。

## 刷新行为

- 应用启动时立即读取一次。
- 悬浮条可见时每 5 分钟自动刷新。
- 悬浮条因吸附的 Codex 窗口隐藏而不可见时暂停周期刷新；重新显示时立即刷新并恢复五分钟节奏。
- 手动拖离 Codex、仍保持可见的悬浮条继续正常刷新。
- 保留手动刷新按钮。
- 已有数据时刷新不切回首次加载状态，只显示按钮忙碌状态。
- 到达 `resetsAt` 时立即刷新，不等待下一次五分钟轮询。
- 自动、手动、重新显示和到点刷新共享一个进行中的请求，不允许并发启动多个 App Server。
- 如果到点时已有普通刷新在进行，先复用该请求；响应仍未进入新周期时再补充刷新一次。

连接失败、未登录或读取超时后，依次在 30 秒、1 分钟和 5 分钟后重试，之后维持每 5 分钟；成功后重置退避。

“未找到七天额度”和“额度数据异常”表示接口调用已经完成，不做快速重试，维持每 5 分钟。

连续刷新失败时，旧数据可保留到当前周期结束，但从首次失败开始必须标记为陈旧。周期结束后，它只能作为上一周期数据展示。

额度快照首版不持久化；“保留旧数据”仅限本次应用运行期间。

## 非目标

- 不展示各个 `rateLimitsByLimitId` 分类额度。
- 不展示原始 token 数；Codex 当前提供的是已用百分比。
- 不把七天额度描述为自然周额度。
- 不把可重置次数描述为已经完成的重置历史。
- 不在本次工作中重新设计设置、更新、拖动或吸附功能。
